## Why the split

The code is open because the product's core claim is that your project data
never leaves your browser. Publishing the source is how that claim can be
verified rather than merely asserted.

The content is a different thing: it is the result of original research and
writing, and it carries no verification value. Licensing it under MIT — as
an unscoped repository licence would — would grant anyone the right to
"use, copy, modify, merge, publish, distribute, sublicense, and/or sell"
it. That was never the intent.
