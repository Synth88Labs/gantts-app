How to Make a Gantt Chart in Google Sheets

By the gantts.app teamUpdated July 14, 20269 min read

There are three practical ways to make a Gantt chart in Google Sheets: use the built-in Timeline feature, build a stacked bar chart with two helper columns, or start from a free template. This guide walks through all three, shows how to customize the view, and explains where Sheets runs out of road — plus a faster, purpose-built alternative that adds dependencies and critical path automatically.

Can Google Sheets make a Gantt chart? (3 methods)

Google Sheets does not offer a dedicated "Gantt chart" chart type the way it offers a pie or line chart. But it gives you three legitimate paths to a working project timeline, each with different trade-offs:

  • The Timeline feature — a native view (Insert → Timeline) that turns a task table into Gantt-style cards on a date axis. Fastest to set up, but it is a view, not an editable chart image, and it does no scheduling math.
  • A stacked bar chart — the classic spreadsheet trick. You add two helper columns and make the first bar segment invisible, leaving a floating "duration" bar for each task. More control, more fiddly.
  • A template — open a ready-made sheet where the formulas and formatting already exist and just type over the sample data.

None of these methods calculate the critical path or draw dependency arrows, and none recalculate downstream dates when one task slips. That gap is the main reason many teams graduate from Sheets to a dedicated tool — more on that below. First, the methods.

Method 1: Insert → Timeline feature

The Timeline view is the quickest way to get a Gantt-style layout in Google Sheets, and it is built in — no formulas required.

  1. Put your data in a simple table with, at minimum, a task/card title column, a start date column, and an end date (or duration) column. Optional columns like owner, group, and % done can feed the card details.
  2. Select the range (including headers).
  3. Go to Insert → Timeline. In the dialog, confirm the data range and click Create.
  4. A new Timeline tab appears. In the right-hand settings panel, map your columns: Card title, Start date, End date, Card color, and Group by.

You can zoom between days, weeks, months, quarters and years, drag to scroll, and click a card to jump to its source row. It is genuinely useful for roadmaps. The catch: a Timeline is an interactive view tied to its source table, not a chart you can drop into a slide as an image, and it has no dependency links or critical-path highlighting.

Method 2: build a stacked bar chart

If you want an actual chart object — one you can copy into Docs or Slides, restyle, and export — the stacked bar chart method is the reliable classic. The idea: each task gets a transparent "spacer" bar that pushes it to its start position, followed by a visible "duration" bar. Stacked together, the visible bars float at the right spot and read as a Gantt chart.

Add Start-on-Day & Duration helper columns

Set up your table with tasks in column A and real dates beside them, then add two calculated columns:

  • Start Date (column B) and End Date (column C) — type the actual calendar dates.
  • Start on Day (column D) — how many days after the project begins each task starts. If the earliest project date is in $B$2, use =B2-$B$2 and format the column as a plain Number (not a date).
  • Duration (column E) — the task length in days: =C2-B2+1. Also format as Number. The +1 counts the end day itself; leave it out and a task that starts and finishes on the same date has a length of zero and draws no bar at all.

The Start on Day column becomes your invisible spacer; Duration becomes the visible bar. Formatting these two columns as numbers rather than dates is the step people most often miss — leave them as dates and Sheets plots two enormous serial numbers and your bars vanish off the right-hand edge.

Insert & format the chart

  1. Select the Task, Start on Day, and Duration columns (A, D and E — hold Ctrl/Cmd to select non-adjacent columns).
  2. Go to Insert → Chart. In the Chart editor, set Chart type to Stacked bar chart.
  3. Click the Start on Day series and set its fill color to None (transparent). The spacer disappears and the Duration bars float into position.
  4. Open Customize → Chart & axis titles and, under the vertical axis, tick Reverse axis order so your first task sits at the top.
  5. Recolor the Duration series, add a chart title, and tidy the horizontal axis so it shows readable dates or day numbers.

You now have an editable Gantt chart you can resize, restyle, and copy elsewhere. Adding milestones or grouping by phase means more helper columns and more manual upkeep, which is where this method starts to feel heavy for real projects.

A worked example: six tasks, then a slip

Helper columns are easy to describe and easy to get subtly wrong. Here is one small plan with real formulas, and then changed — because how a sheet behaves when a date moves is the only test that matters.

The sheet. A website replatform starting Monday 2 March 2026. Headers in row 1, tasks in rows 2–7. A Task, B Start, C End, D Working days, E Start on day, F Bar length. Only column D is typed by hand:

  • C2 — =WORKDAY(B2,D2-1). Sheets has the same WORKDAY as Excel, and it skips weekends for you. The -1 is essential: WORKDAY counts forward from the start, so five working days from Monday is WORKDAY(Mon,4) = Friday.
  • B3 — =WORKDAY(C2,1). The next working day after the predecessor ends. This is the only thing in the file that reschedules anything.
  • E2 — =B2-$B$2, formatted as Number. Zero for the first task.
  • F2 — =C2-B2+1, formatted as Number. Note this is not D2. D is working days; the chart axis is calendar days, so a 10-working-day task must draw 12 units long. Plot D by mistake and every bar finishes early while the table beside it reads correctly.

Fill down and you get:

RowTaskB — StartC — EndD — Working daysE — Start on dayF — Bar length
2DiscoveryMon 2 MarFri 13 Mar10012
3DesignMon 16 MarFri 27 Mar101412
4BuildMon 30 MarFri 24 Apr202826
5Content migrationMon 27 AprFri 15 May155619
6QAMon 18 MayFri 29 May107712
7Go liveMon 1 JunMon 1 Jun1911

Sanity check the last row: 91 + 1 = 92 days, the project's full span from 2 March. If E + F on the bottom row is not the span you expect, a convention has drifted somewhere above.

Now change something. Design overruns by 3 days: edit D3 from 10 to 13. The WORKDAY chain re-computes and everything downstream moves — Design ends Wed 1 Apr, Build runs Thu 2 Apr to Wed 29 Apr, Content migration Thu 30 Apr to Wed 20 May, QA Thu 21 May to Wed 3 Jun, go-live Thu 4 Jun. Both helper columns follow on their own. That is the reason to use formulas rather than typed dates.

And here is what does not.

  1. Inserted rows arrive without formulas. Add "Client review" between Design and Build: the new row 4 is blank, and old Build — now row 5 — still points at Design's End. Sheets extends the chart range regardless, so the empty row plots as a zero-length bar at day zero, a stub at the far left that reads as a rendering glitch rather than a mistake.
  2. Sorting destroys the chain. Sort by start date and every relative reference follows its new neighbour, quietly re-deriving dates from the wrong rows. No warning, and no undo once you have saved and closed.
  3. A second predecessor has nowhere to live. If QA needs both Build and Content migration, the correct formula is =WORKDAY(MAX(C4,C5),1). It works. But nothing on the chart shows the link, and a five-day client review becomes =WORKDAY(C3,1+5) — a bare 5 inside a formula, labelled nowhere.

The Sheets-only shortcut. If you want a bar per row without a chart object at all, put this in G2 and fill down: =SPARKLINE({E2,F2},{"charttype","bar";"color1","white";"color2","#6c4cf1";"max",92}). It draws the same invisible-spacer trick inside a single cell, and because max is fixed at the project span every row shares one axis. It survives inserted rows and re-sorting better than the chart does — but it is still a picture, not a schedule.

Customize the view & add dependencies

A few tweaks make any Sheets Gantt chart more readable:

  • Group by phase — add a "Phase" column and either sort by it or use the Timeline's Group by field to cluster related tasks.
  • Show progress — add a % complete column and, in the stacked-bar method, split each duration bar into "done" and "remaining" segments for a shaded progress look.
  • Mark milestones — give zero-duration tasks a distinct color so they read as points rather than bars.
  • Add a today line — a single-value series or a vertical reference line helps readers see status at a glance.

Dependencies are the hard limit. Google Sheets has no concept of "Task B starts when Task A finishes." You can fake the finish-to-start case with =WORKDAY(C2,1), as above, but there are no visible arrows, the other three link types have no expression at all, and complex chains become brittle. Frequently asked questions

Does Google Sheets have a built-in Gantt chart?

Google Sheets does not have a one-click Gantt chart type, but it includes a Timeline view (Insert → Timeline) that produces a Gantt-style bar layout from a table, and you can build a true Gantt chart with a stacked bar chart. Neither supports dependencies or a critical path.

What is the Timeline feature in Google Sheets?

Timeline is a view added to Google Sheets that turns a table with start and end dates into horizontal cards on a time axis. It is quick to set up and good for roadmaps, but it is not editable like a chart image and does not calculate dependencies or critical path.

How do I add dependencies to a Google Sheets Gantt chart?

Google Sheets cannot draw dependency arrows or recalculate dates automatically. You would have to adjust each task's dates by hand. For automatic dependencies and critical path, use a dedicated Gantt tool such as gantts.app, which is free and needs no sign up.

Why are my Google Sheets Gantt bars in the wrong place?

Almost always one of two things. Either the Start-on-Day and Duration helper columns are still formatted as dates, so Sheets plots two very large serial numbers and the bars run off the right-hand edge — format both as plain Number. Or the chart is plotting working days instead of calendar days: the horizontal axis counts every day including weekends, so a ten-working-day task must have a bar length of twelve, computed as =C2-B2+1.

Can I make a Gantt chart in a single Google Sheets cell?

Yes, with SPARKLINE. With a Start-on-Day value in E2 and a bar length in F2, =SPARKLINE({E2,F2},{"charttype","bar";"color1","white";"color2","#6c4cf1";"max",92}) draws a bar inside the cell. Setting max to the project span in days keeps every row on the same scale. It handles inserted and re-sorted rows better than a chart object, but it still has no dependencies or critical path.

Can I make a Gantt chart in Google Sheets for free?

Yes. Google Sheets is free with a Google account, and all three methods — the Timeline view, a stacked bar chart, and a template — cost nothing. gantts.app is also free and requires no account at all if you want a purpose-built Gantt editor.

How do I convert a Google Sheets Gantt chart to Excel or PowerPoint?

You can download a Sheet as .xlsx via File → Download, or copy a chart image into slides. A faster route is to build the chart in gantts.app and export directly to Excel, PowerPoint, PDF or PNG in one click.
