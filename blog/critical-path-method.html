Critical Path Method (CPM): Steps, Formula & Example

By the gantts.app teamUpdated July 14, 202611 min read

The critical path method (CPM) finds the longest chain of dependent tasks in a project. That chain sets the shortest possible finish date, because its tasks have no slack — delay any one of them and the whole project slips. This guide explains the terms, the forward and backward pass, and a worked example with the math.

What is CPM?

The critical path method is a project scheduling technique that identifies the longest sequence of dependent tasks from start to finish. The length of that sequence equals the shortest time in which the project can be completed. The tasks on it are called critical tasks because they have zero slack: there is no spare time, so any delay to a critical task pushes out the project's end date one-for-one.

CPM was developed in the late 1950s (by Morgan Walker and James Kelley, working on plant-maintenance scheduling) and remains one of the most widely taught tools in project management. Its value is focus: on a plan with dozens of tasks, CPM tells you exactly which handful actually control the deadline, so you know where to concentrate attention, resources, and risk management.

Key terms

CPM has its own vocabulary. A few definitions make the calculation easy to follow:

  • Critical path — the longest path of dependent activities through the project; its duration is the project's minimum completion time.
  • Float (slack) — how long an activity can be delayed without delaying the project finish. Critical activities have zero float.
  • Forward pass — a left-to-right calculation through the network that finds each activity's earliest possible dates.
  • Backward pass — a right-to-left calculation that finds each activity's latest possible dates without slipping the end date.
  • ES / EF (Early Start / Early Finish) — the earliest an activity can start and finish. EF = ES + Duration.
  • LS / LF (Late Start / Late Finish) — the latest an activity can start and finish without delaying the project. LS = LF − Duration.

The core formula that ties it together is Float = LS − ES = LF − EF. When float is zero, the activity is on the critical path.

How to calculate it step by step

CPM is a five-step procedure. We will use a day-based convention where the project starts at time 0, so the first activity has ES = 0.

  1. List activities & dependencies. Write down every activity, its duration, and which activities must finish before it can start (its predecessors).
  2. Build the network. Draw the activities as a network diagram connected by their dependencies, from a single start to a single finish. This shows every possible path through the project.
  3. Forward pass (ES / EF). Moving left to right, set the first activity's ES = 0 and compute EF = ES + Duration. Each following activity's ES is the largest EF among its predecessors; carry EF forward. The largest EF at the end is the project duration.
  4. Backward pass (LS / LF). Moving right to left, set the last activity's LF equal to the project duration and compute LS = LF − Duration. Each earlier activity's LF is the smallest LS among the activities that follow it.
  5. Compute float & identify the path. For each activity, Float = LS − ES. The activities with zero float, connected start to finish, are the critical path.

A worked example

Consider a small project with six activities. Durations are in days; predecessors show what must finish first.

A — Plan3—
B — Design4A
C — Procure2A
D — Build5B
E — Prepare site6C
F — Install & test3D, E

There are two paths from start to finish: A → B → D → F = 3 + 4 + 5 + 3 = 15 days, and A → C → E → F = 3 + 2 + 6 + 3 = 14 days. The longer one, 15 days, is the critical path — but let us confirm it with the full forward and backward pass and see the float on every activity.

Forward pass (ES, then EF = ES + Duration):

  • A: ES 0, EF 3
  • B: ES 3 (after A), EF 7
  • C: ES 3 (after A), EF 5
  • D: ES 7 (after B), EF 12
  • E: ES 5 (after C), EF 11
  • F: ES 12 (max of D's EF 12 and E's EF 11), EF 15 → project = 15 days

Backward pass (LF, then LS = LF − Duration), starting from F's LF = 15:

  • F: LF 15, LS 12
  • D: LF 12 (F's LS), LS 7
  • E: LF 12 (F's LS), LS 6
  • B: LF 7 (D's LS), LS 3
  • C: LF 6 (E's LS), LS 4
  • A: LF 3 (min of B's LS 3 and C's LS 4), LS 0

Float = LS − ES for each activity:

ActivityESEFLSLFFloatCritical?
A03030Yes
B37370Yes
C35461No
D7127120Yes
E5116121No
F121512150Yes

The activities with zero float — A, B, D, F — form the critical path, exactly matching the longest path we spotted earlier. Activities C and E each have 1 day of float, meaning either could slip by one day without delaying the 15-day finish. Push C or E two days, though, and E's path (now 15) becomes critical too. That is the practical payoff of CPM: it tells you which tasks to guard (the zero-float ones) and how much wiggle room the rest have.

CPM vs PERT vs critical chain

Three related techniques are easy to mix up:

  • CPM uses a single, fixed duration per activity and focuses on the critical path and float. Best when durations are reasonably well known.
  • PERT (Program Evaluation and Review Technique) uses three estimates per activity — optimistic (O), most likely (M), and pessimistic (P) — and a weighted average, usually (O + 4M + P) / 6, to model uncertainty and estimate the probability of hitting a date. Best when durations are uncertain. In practice teams often run CPM on PERT-derived estimates.
  • Critical chain (from the theory of constraints) extends CPM by adding resource constraints and consolidating individual task padding into shared "buffers" placed at the end of the chain and its feeders. Best when shared resources, not just dependencies, drive the schedule.

Frequently asked questions

What is the critical path method in simple terms?

The critical path method is a way to find the longest chain of dependent tasks in a project. That chain determines the shortest possible finish date, because every task on it must be done in sequence with no slack. Delaying any critical task delays the whole project.

How do you calculate the critical path?

List every activity with its duration and dependencies, draw a network diagram, run a forward pass to find each task's earliest start and finish, run a backward pass to find the latest start and finish, then compute float as late start minus early start. Tasks with zero float form the critical path.

What is float or slack?

Float, also called slack, is how long a task can be delayed without delaying the project. It equals late start minus early start (or late finish minus early finish). Tasks on the critical path have zero float; non-critical tasks have positive float.

What is the difference between the forward pass and backward pass?

The forward pass moves left to right through the network to calculate each task's earliest start and earliest finish, giving the project's minimum duration. The backward pass moves right to left to calculate each task's latest start and latest finish without delaying that end date.

What is the difference between CPM and PERT?

CPM uses a single, deterministic duration for each activity and focuses on the critical path and float. PERT uses three time estimates (optimistic, most likely, pessimistic) to model uncertainty and estimate the probability of finishing by a given date. In practice the two are often combined.
