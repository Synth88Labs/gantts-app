Modelo de cronograma para projeto de software

Um cronograma de projeto de software tem uma armadilha recorrente: a homologação é planejada como uma barra curta no fim e leva o dobro. Teste e correção formam um ciclo, não uma linha reta.

O que o modelo contém

Seis fases, dos requisitos à produção:

  • Requisitos — Necessidades, casos de uso, critérios de aceite. Marco: escopo congelado.
  • Arquitetura — Decisões técnicas, modelo de dados, integrações, ambientes.
  • Desenvolvimento — Execução em iterações, revisão de código, integração contínua.
  • Integração e homologação — Montagem, testes funcionais, correções — em ciclo com o desenvolvimento.
  • Migração de dados — Extração, mapeamento, migrações de ensaio. Várias rodadas, nunca uma só.
  • Entrada em produção — Virada, monitoramento reforçado, estabilização. Marco: em produção.
Conte com várias migrações de ensaio, não com uma. A qualidade dos dados existentes é quase sempre pior do que o previsto, e cada rodada revela casos ausentes da especificação. Uma única migração na noite da virada é a forma clássica de errar uma entrada em produção.

Como adaptar

  1. Acrescente uma linha por módulo ou por equipe se houver trabalho em paralelo.
  2. Represente a homologação em ciclo com o desenvolvimento, não em sequência.
  3. Preveja uma fase de estabilização após a virada — de duas a seis semanas.
  4. Coloque o congelamento de escopo como marco: o que entra depois empurra a produção.

Dicas de cronograma

  • A homologação leva o dobro do previsto. Planeje assim.
  • Teste a migração cedo, com dados reais.
  • Evite virada na sexta-feira ou em período de fechamento.
  • Estabilização é fase, não imprevisto.

Perguntas frequentes

O Gantt serve para projeto ágil?

Para o sprint, o quadro continua sendo a ferramenta. O Gantt serve para marcos, dependências externas e alinhamento com áreas não ágeis.

Quando congelar o escopo?

Antes da arquitetura. Qualquer acréscimo posterior empurra a produção, e isso precisa ficar visível.

Quantas migrações de ensaio?

De três a cinco conforme a qualidade dos dados atuais. Uma só nunca basta.
